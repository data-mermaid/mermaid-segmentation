# MERMAID Concept Bottleneck Demo

Interactive Gradio demo for concept-bottleneck coral segmentation models (linear CBM, DPT, DPT+LoRA). Model definitions and concept helpers are imported from the `mermaidseg` package (not vendored in this folder).

## Artifacts / provenance

The demo needs a checkpoint plus label/concept metadata. Most of this is **not generated by a standalone script** — it comes from a training run and is logged to MLflow:

| File | MLflow artifact path | How it is produced |
|------|---------------------|-------------------|
| `id2label.json` | `metadata/id2label.json` | Built in `scripts/train.py` as `{0: "ignore", **registry.target_id2label}` and logged by `Logger` |
| `concept_id2name.json` | `metadata/concept_id2name.json` | Built by `SourceLabelRegistry._build_concepts()` and logged via `Logger.log_reconciliation()` |

Download both from an MLflow run (under `metadata/`) and place them in `demo/`, or pass paths with `--id2label` / `--concept-id2name`.

The taxonomy tree uses `configs/class_to_concepts.csv` from the repo (or any compatible CSV). Set `DEMO_TAXONOMY_CSV` to override; if the file is missing, the demo still runs but taxonomy parent edges are omitted.

## Local usage (repo environment)

From the repo root:

```bash
uv sync --extra demo
export DEMO_CHECKPOINT=/path/to/model_checkpoint
export DEMO_MODEL_CONFIG=configs/model_config_cbm_dpt_lora_vitl.yaml
# optional: export DEMO_TAXONOMY_CSV=configs/class_to_concepts.csv
uv run python demo/app.py --port 7860
```

Or pass paths explicitly:

```bash
uv run python demo/app.py \
  --checkpoint /path/to/checkpoint \
  --model-config configs/model_config_cbm_dpt_lora_vitl.yaml \
  --taxonomy-csv configs/class_to_concepts.csv
```

Supported model configs (`model.name` in the YAML):

- `ConceptBottleneckDINOv3` — linear head CBM (`model_config_cbm_vitl.yaml`)
- `ConceptBottleneckDPTDINOv3` — frozen ViT-L + DPT head (`model_config_cbm_dpt_vitl.yaml`)
- `ConceptBottleneckDPTLoRADINOv3` — LoRA-adapted ViT-L + DPT head (`model_config_cbm_dpt_lora_vitl.yaml`)

## Deploy to Hugging Face Spaces

`gradio deploy` uploads only the `demo/` folder. The package is installed from GitHub via [requirements.txt](requirements.txt):

```bash
cd demo
gradio deploy
```

Set Space secrets (or env vars):

- `DEMO_CHECKPOINT` — path or URL to the checkpoint inside the Space
- `DEMO_MODEL_CONFIG` — path to model config YAML (bundle a copy in `demo/` if needed)
- `DEMO_TAXONOMY_CSV` — optional; defaults to repo `configs/class_to_concepts.csv` when running from a full clone

Pin the `mermaidseg` git ref in `requirements.txt` (`@main` or a release tag) so Spaces pick up a known package version.

## Features

- Dual **one-hot** (MERMAID class or taxonomic rank) and **multi-hot** (morphologic/non-coral concept) overlay panels
- Click a pixel to inspect top classes, taxonomy tree, and other concept activations
- Sample image gallery from `static/`
